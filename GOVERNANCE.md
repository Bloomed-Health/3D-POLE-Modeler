# Governance

## Project Overview

3D POLE Modeler is maintained by **Bloomed Health**. This document describes how the project is governed and how decisions are made.

## Roles

### Maintainers

Maintainers have full commit access and are responsible for:

- Reviewing and merging pull requests
- Triaging issues and setting priorities
- Releasing new versions
- Enforcing the Code of Conduct

Current maintainers:

- Charlie C ([@charliec](https://github.com/charliec)) — Project Lead

### Contributors

Anyone who submits a pull request, files an issue, or participates in discussions is a contributor. Contributors are expected to follow the [Contributing Guidelines](CONTRIBUTING.md).

## Decision-Making

- **Day-to-day decisions** (bug fixes, minor enhancements, documentation) are made by individual maintainers through the normal pull request review process.
- **Significant changes** (new features, architectural changes, dependency additions, API changes) require approval from the Project Lead.
- **Governance changes** (this document, licensing, Code of Conduct) require approval from the Project Lead.

## Contributions

All contributions must comply with the [Developer Certificate of Origin (DCO)](https://developercertificate.org/). See [CONTRIBUTING.md](CONTRIBUTING.md) for details.

## Conflict Resolution

If a disagreement cannot be resolved through discussion on an issue or pull request, the Project Lead has final decision-making authority.

## Changes to Governance

This governance model may evolve as the project grows. Proposed changes should be submitted as a pull request to this file.
